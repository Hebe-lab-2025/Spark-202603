# AWS CDK 里 L1 / L2 / L3 Constructs 代码对比 Demo

我用最常见的思路讲：

* **L1**：最底层，接近 CloudFormation
* **L2**：最常用，面向对象封装
* **L3**：更高层，直接给你一整套模式

---

# 1. 先记一句最重要的话

## Construct 是什么？

Construct 就是 **CDK 里的“积木”**。
你用这些积木去搭 AWS 资源。

比如：

* S3 Bucket
* Lambda Function
* API Gateway
* DynamoDB Table

都可以用 construct 表示。

---

# 2. L1 Construct

L1 基本就是 **CloudFormation resource 的代码版**。

命名一般长这样：

* `CfnBucket`
* `CfnFunction`
* `CfnTable`

## 例子：L1 创建 S3 Bucket

```ts
import * as cdk from 'aws-cdk-lib';
import * as s3 from 'aws-cdk-lib/aws-s3';
import { Construct } from 'constructs';

export class MyL1Stack extends cdk.Stack {
  constructor(scope: Construct, id: string) {
    super(scope, id);

    new s3.CfnBucket(this, 'MyL1Bucket', {
      bucketName: 'my-l1-demo-bucket'
    });
  }
}
```

## 解释

这里的：

```ts
s3.CfnBucket
```

就是 **L1 construct**。

它几乎直接对应 CloudFormation 里的 `AWS::S3::Bucket`。

### 特点

* 很底层
* 属性名更像 CloudFormation
* 更灵活
* 但不好写

---

# 3. L2 Construct

L2 是对单个 AWS 资源做了更友好的封装。
这是平时最常用的。

## 例子：L2 创建 S3 Bucket

```ts
import * as cdk from 'aws-cdk-lib';
import * as s3 from 'aws-cdk-lib/aws-s3';
import { Construct } from 'constructs';

export class MyL2Stack extends cdk.Stack {
  constructor(scope: Construct, id: string) {
    super(scope, id);

    new s3.Bucket(this, 'MyL2Bucket', {
      versioned: true,
      encryption: s3.BucketEncryption.S3_MANAGED
    });
  }
}
```

## 解释

这里的：

```ts
s3.Bucket
```

就是 **L2 construct**。

它比 L1 好用很多，因为：

* API 更像开发者写代码的风格
* 帮你封装了一些细节
* 默认值更合理

### 你会发现

L1 写法更“原始”，
L2 写法更“工程化”。

---

# 4. L1 和 L2 对比理解

## L1

你自己和 AWS 底层细节打交道。

## L2

CDK 已经帮你包装了一层。

所以面试可以说：

> L1 constructs are direct CloudFormation mappings, while L2 constructs are higher-level abstractions that make resource creation easier and more developer-friendly.

---

# 5. L3 Construct

L3 不是只代表一个资源。

它通常是 **多个资源组合起来的高层模式**。

比如：

* Lambda + API Gateway
* Lambda + S3 notification
* VPC + ECS + Load Balancer

它更像“解决方案模板”。

---

# 6. L3 例子：LambdaRestApi

这个特别经典。

```ts
import * as cdk from 'aws-cdk-lib';
import * as lambda from 'aws-cdk-lib/aws-lambda';
import * as apigateway from 'aws-cdk-lib/aws-apigateway';
import { Construct } from 'constructs';

export class MyL3Stack extends cdk.Stack {
  constructor(scope: Construct, id: string) {
    super(scope, id);

    const myFunction = new lambda.Function(this, 'MyFunction', {
      runtime: lambda.Runtime.NODEJS_18_X,
      handler: 'index.handler',
      code: lambda.Code.fromInline(`
        exports.handler = async function(event) {
          return {
            statusCode: 200,
            body: "Hello from Lambda"
          };
        };
      `),
    });

    new apigateway.LambdaRestApi(this, 'MyApi', {
      handler: myFunction
    });
  }
}
```

## 为什么这个算高层？

因为：

```ts
new apigateway.LambdaRestApi(...)
```

它背后不只是一个单独资源。

它通常会帮你处理：

* API Gateway
* Lambda integration
* permission
* deployment/stage 的一些配置

所以它不是一个“单资源封装”，
而是一个“组合模式”。

# 8. 再给你一个更直观的对比

## 用 L1 创建 Lambda（更底层）

```ts
new lambda.CfnFunction(this, 'MyL1Lambda', {
  functionName: 'my-l1-lambda',
  role: 'arn:aws:iam::123456789012:role/service-role/my-role',
  runtime: 'nodejs18.x',
  handler: 'index.handler',
  code: {
    zipFile: `
      exports.handler = async function(event) {
        return { statusCode: 200, body: "hello" };
      };
    `
  }
});
```

### 问题

这里你要自己处理很多细节，比如：

* IAM role
* code format
* 底层属性写法

---

## 用 L2 创建 Lambda（更常用）

```ts
new lambda.Function(this, 'MyL2Lambda', {
  runtime: lambda.Runtime.NODEJS_18_X,
  handler: 'index.handler',
  code: lambda.Code.fromInline(`
    exports.handler = async function(event) {
      return { statusCode: 200, body: "hello" };
    };
  `),
});
```

### 好处

* 类型更安全
* API 更清晰
* 更容易读写

# 9. 三者本质区别

| 层级 | 代表什么                 | 粒度  | 特点       |
| -- | -------------------- | --- | -------- |
| L1 | 原始 CloudFormation 资源 | 最底层 | 完全控制，但啰嗦 |
| L2 | 单个 AWS 资源高级封装        | 中间层 | 最常用，易开发  |
| L3 | 多资源组合模式              | 最高层 | 快速搭业务方案  |

# 10. 面试标准回答

你可以直接背这个：

> In AWS CDK, constructs are reusable building blocks for infrastructure as code.
> L1 constructs are the lowest-level constructs and directly map to CloudFormation resources.
> L2 constructs are higher-level abstractions for individual AWS services, making them easier to use.
> L3 constructs are even higher-level patterns that combine multiple resources into complete solutions.

# 11. 什么时候用哪一种？

## 用 L1

当你需要：

* very fine-grained control
* access to newly released AWS features
* direct CloudFormation-level properties

## 用 L2

当你需要：

* normal day-to-day development
* readable and maintainable code
* simpler APIs

## 用 L3

当你需要：

* faster delivery
* common architecture patterns
* less boilerplate code

# 8. 第三步：写 Lambda 代码

文件路径：

```text
services/lambda/index.py
```

代码如下：

```python
import json
import os
import boto3
from urllib.parse import unquote_plus

dynamodb = boto3.resource('dynamodb')
table_name = os.environ['TABLE_NAME']
table = dynamodb.Table(table_name)

def lambda_handler(event, context):
    print("Received event:", json.dumps(event))

    for record in event['Records']:
        bucket_name = record['s3']['bucket']['name']
        object_key = unquote_plus(record['s3']['object']['key'])
        file_size = record['s3']['object'].get('size', 0)

        item = {
            'fileName': object_key,
            'bucketName': bucket_name,
            'fileSize': file_size
        }

        table.put_item(Item=item)

    return {
        'statusCode': 200,
        'body': json.dumps('File metadata saved to DynamoDB')
    }
```

# 9. 第四步：写 CDK Stack

文件路径：

```text
infra/lib/infra-stack.ts
```

把内容改成下面这个：

```ts
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as s3 from 'aws-cdk-lib/aws-s3';
import * as lambda from 'aws-cdk-lib/aws-lambda';
import * as dynamodb from 'aws-cdk-lib/aws-dynamodb';
import * as s3n from 'aws-cdk-lib/aws-s3-notifications';

export class InfraStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    // Create DynamoDB table
    const table = new dynamodb.Table(this, 'FileMetadataTable', {
      partitionKey: {
        name: 'fileName',
        type: dynamodb.AttributeType.STRING,
      },
      billingMode: dynamodb.BillingMode.PAY_PER_REQUEST,
      removalPolicy: cdk.RemovalPolicy.DESTROY,
    });

    // Create S3 bucket
    const bucket = new s3.Bucket(this, 'UploadBucket', {
      removalPolicy: cdk.RemovalPolicy.DESTROY,
      autoDeleteObjects: true,
    });

    // Create Lambda function
    const fileProcessorFunction = new lambda.Function(this, 'FileProcessorFunction', {
      runtime: lambda.Runtime.PYTHON_3_11,
      handler: 'index.lambda_handler',
      code: lambda.Code.fromAsset('../services/lambda'),
      environment: {
        TABLE_NAME: table.tableName,
      },
    });

    // Grant Lambda permission to write to DynamoDB
    table.grantWriteData(fileProcessorFunction);

    // Add S3 event notification to trigger Lambda on upload
    bucket.addEventNotification(
      s3.EventType.OBJECT_CREATED,
      new s3n.LambdaDestination(fileProcessorFunction)
    );

    // Grant S3 permission to invoke Lambda automatically
    fileProcessorFunction.addPermission('AllowS3InvokeLambda', {
      principal: new cdk.aws_iam.ServicePrincipal('s3.amazonaws.com'),
      sourceArn: bucket.bucketArn,
    });

    // Outputs
    new cdk.CfnOutput(this, 'BucketName', {
      value: bucket.bucketName,
    });

    new cdk.CfnOutput(this, 'TableName', {
      value: table.tableName,
    });
  }
}
```

## `lib/cdk-demo-stack.ts`

```ts
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as s3 from 'aws-cdk-lib/aws-s3';
import * as lambda from 'aws-cdk-lib/aws-lambda';
import * as apigateway from 'aws-cdk-lib/aws-apigateway';

export class CdkDemoStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    // Create an S3 bucket to store processed results
    const resultBucket = new s3.Bucket(this, 'ResultBucket', {
      versioned: true,
      removalPolicy: cdk.RemovalPolicy.DESTROY,
      autoDeleteObjects: true
    });

    // Create a Lambda function
    const processFunction = new lambda.Function(this, 'ProcessFunction', {
      runtime: lambda.Runtime.NODEJS_18_X,
      handler: 'index.handler',
      code: lambda.Code.fromAsset('lambda'),
      environment: {
        BUCKET_NAME: resultBucket.bucketName
      }
    });

    // Grant Lambda permission to write to S3
    resultBucket.grantPut(processFunction);

    // Create API Gateway and connect it to Lambda
    new apigateway.LambdaRestApi(this, 'ProcessApi', {
      handler: processFunction,
      proxy: true
    });

    // Output the bucket name
    new cdk.CfnOutput(this, 'BucketNameOutput', {
      value: resultBucket.bucketName
    });
  }
}
```

## `lambda/index.js`

```js
const AWS = require('aws-sdk');
const s3 = new AWS.S3();

exports.handler = async (event) => {
  try {
    // Parse request body
    const body = event.body ? JSON.parse(event.body) : {};
    const message = body.message || 'default message';

    // Create file content
    const content = `Processed message: ${message}\nTimestamp: ${new Date().toISOString()}`;

    // Generate a unique file name
    const fileName = `result-${Date.now()}.txt`;

    // Write result to S3
    await s3.putObject({
      Bucket: process.env.BUCKET_NAME,
      Key: fileName,
      Body: content,
      ContentType: 'text/plain'
    }).promise();

    // Return success response
    return {
      statusCode: 200,
      body: JSON.stringify({
        message: 'File processed successfully',
        fileName: fileName,
        bucket: process.env.BUCKET_NAME
      })
    };
  } catch (error) {
    return {
      statusCode: 500,
      body: JSON.stringify({
        message: 'Processing failed',
        error: error.message
      })
    };
  }
};
```

# 5. 第三步：写 Lambda 代码

文件位置：

```text

```

代码如下：

```python
import json
import os
import boto3
from urllib.parse import unquote_plus

# Create DynamoDB resource
dynamodb = boto3.resource("dynamodb")

# Read table name from Lambda environment variable
TABLE_NAME = os.environ["TABLE_NAME"]

# Get DynamoDB table reference
table = dynamodb.Table(TABLE_NAME)


def lambda_handler(event, context):
    """
    This function is triggered by S3 upload events.
    It reads uploaded file metadata and stores it in DynamoDB.
    """

    print("Received event:")
    print(json.dumps(event))

    records = event.get("Records", [])

    for record in records:
        # Get bucket name
        bucket_name = record["s3"]["bucket"]["name"]

        # Get uploaded object key (file name / path)
        object_key = unquote_plus(record["s3"]["object"]["key"])

        # Get file size if available
        file_size = record["s3"]["object"].get("size", 0)

        # Store metadata into DynamoDB
        table.put_item(
            Item={
                "fileName": object_key,
                "bucketName": bucket_name,
                "fileSize": file_size
            }
        )

    return {
        "statusCode": 200,
        "body": json.dumps("File metadata saved successfully")
    }
```

# 6. 第四步：写 CDK Stack

文件位置：

```text
aws-cdk-file-processor/infra/lib/infra-stack.ts
```

把原来的内容替换成下面：

```ts
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as s3 from 'aws-cdk-lib/aws-s3';
import * as lambda from 'aws-cdk-lib/aws-lambda';
import * as dynamodb from 'aws-cdk-lib/aws-dynamodb';
import * as s3n from 'aws-cdk-lib/aws-s3-notifications';

export class InfraStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    // Create DynamoDB table to store file metadata
    const table = new dynamodb.Table(this, 'FileMetadataTable', {
      partitionKey: {
        name: 'fileName',
        type: dynamodb.AttributeType.STRING,
      },
      billingMode: dynamodb.BillingMode.PAY_PER_REQUEST,
      removalPolicy: cdk.RemovalPolicy.DESTROY,
    });

    // Create S3 bucket for file uploads
    const bucket = new s3.Bucket(this, 'UploadBucket', {
      removalPolicy: cdk.RemovalPolicy.DESTROY,
      autoDeleteObjects: true,
    });

    // Create Lambda function
    const fileProcessorFunction = new lambda.Function(this, 'FileProcessorFunction', {
      runtime: lambda.Runtime.PYTHON_3_11,
      handler: 'index.lambda_handler',
      code: lambda.Code.fromAsset('../services/lambda'),
      environment: {
        TABLE_NAME: table.tableName,
      },
    });

    // Allow Lambda to write data to DynamoDB
    table.grantWriteData(fileProcessorFunction);

    // Add S3 event trigger for object creation
    bucket.addEventNotification(
      s3.EventType.OBJECT_CREATED,
      new s3n.LambdaDestination(fileProcessorFunction)
    );

    // Output bucket name
    new cdk.CfnOutput(this, 'BucketName', {
      value: bucket.bucketName,
    });

    // Output DynamoDB table name
    new cdk.CfnOutput(this, 'TableName', {
      value: table.tableName,
    });

    // Output Lambda name
    new cdk.CfnOutput(this, 'LambdaName', {
      value: fileProcessorFunction.functionName,
    });
  }
}
```

## 4. Java Lambda 代码

### `Handler.java`

```java
package com.example;

import com.amazonaws.services.lambda.runtime.Context;
import com.amazonaws.services.lambda.runtime.RequestHandler;
import software.amazon.awssdk.services.dynamodb.DynamoDbClient;
import software.amazon.awssdk.services.dynamodb.model.AttributeValue;
import software.amazon.awssdk.services.dynamodb.model.PutItemRequest;

import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class Handler implements RequestHandler<Map<String, Object>, Response> {

    private final DynamoDbClient dynamoDbClient = DynamoDbClient.create();
    private final String tableName = System.getenv("TABLE_NAME");

    @Override
    public Response handleRequest(Map<String, Object> event, Context context) {
        context.getLogger().log("Received event: " + event);

        try {
            Object recordsObj = event.get("Records");
            if (!(recordsObj instanceof List)) {
                return new Response(400, "No Records found in event");
            }

            List<?> records = (List<?>) recordsObj;

            for (Object recordObj : records) {
                if (!(recordObj instanceof Map)) {
                    continue;
                }

                Map<?, ?> record = (Map<?, ?>) recordObj;
                Map<?, ?> s3 = (Map<?, ?>) record.get("s3");
                Map<?, ?> bucket = (Map<?, ?>) s3.get("bucket");
                Map<?, ?> object = (Map<?, ?>) s3.get("object");

                String bucketName = String.valueOf(bucket.get("name"));
                String fileName = String.valueOf(object.get("key"));
                String fileSize = String.valueOf(object.getOrDefault("size", "0"));

                Map<String, AttributeValue> item = new HashMap<>();
                item.put("fileName", AttributeValue.builder().s(fileName).build());
                item.put("bucketName", AttributeValue.builder().s(bucketName).build());
                item.put("fileSize", AttributeValue.builder().n(fileSize).build());

                PutItemRequest putItemRequest = PutItemRequest.builder()
                        .tableName(tableName)
                        .item(item)
                        .build();

                dynamoDbClient.putItem(putItemRequest);
            }

            return new Response(200, "File metadata saved successfully");
        } catch (Exception e) {
            context.getLogger().log("Error: " + e.getMessage());
            return new Response(500, "Internal Server Error: " + e.getMessage());
        }
    }
}
```

## 7. CDK Stack

### `infra/lib/infra-stack.ts`

```ts
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as s3 from 'aws-cdk-lib/aws-s3';
import * as lambda from 'aws-cdk-lib/aws-lambda';
import * as dynamodb from 'aws-cdk-lib/aws-dynamodb';
import * as s3n from 'aws-cdk-lib/aws-s3-notifications';

export class InfraStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    const table = new dynamodb.Table(this, 'FileMetadataTable', {
      partitionKey: {
        name: 'fileName',
        type: dynamodb.AttributeType.STRING,
      },
      billingMode: dynamodb.BillingMode.PAY_PER_REQUEST,
      removalPolicy: cdk.RemovalPolicy.DESTROY,
    });

    const bucket = new s3.Bucket(this, 'UploadBucket', {
      removalPolicy: cdk.RemovalPolicy.DESTROY,
      autoDeleteObjects: true,
    });

    const fileProcessorFunction = new lambda.Function(this, 'JavaFileProcessorFunction', {
      runtime: lambda.Runtime.JAVA_17,
      handler: 'com.example.Handler::handleRequest',
      code: lambda.Code.fromAsset('../services/lambda-java/target'),
      memorySize: 1024,
      timeout: cdk.Duration.seconds(30),
      environment: {
        TABLE_NAME: table.tableName,
      },
    });

    table.grantWriteData(fileProcessorFunction);

    bucket.addEventNotification(
      s3.EventType.OBJECT_CREATED,
      new s3n.LambdaDestination(fileProcessorFunction)
    );

    new cdk.CfnOutput(this, 'BucketName', {
      value: bucket.bucketName,
    });

    new cdk.CfnOutput(this, 'TableName', {
      value: table.tableName,
    });

    new cdk.CfnOutput(this, 'LambdaName', {
      value: fileProcessorFunction.functionName,
    });
  }
}
```
