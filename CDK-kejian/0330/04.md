## 🧩 这个架构包含什么？

👉 作用：对外提供 HTTP 接口（类似后端入口）

👉 作用：处理业务逻辑

* 返回结果

---

### 3️⃣ Amazon S3（对象存储）

👉 作用：存储数据（文件/图片/JSON等）

📌 类比：仓库 / 数据存储

# 🪜 Step 3：编写 Stack（核心代码）

编辑文件：`lib/cdk-demo-stack.ts`

```ts
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';

// S3
import * as s3 from 'aws-cdk-lib/aws-s3';

// Lambda
import * as lambda from 'aws-cdk-lib/aws-lambda';

// API Gateway
import * as apigw from 'aws-cdk-lib/aws-apigateway';

export class CdkDemoStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    // 1️⃣ 创建 S3 Bucket
    const bucket = new s3.Bucket(this, 'MyBucket', {
      versioned: true
    });

    // 2️⃣ 创建 Lambda
    const fn = new lambda.Function(this, 'MyFunction', {
      runtime: lambda.Runtime.NODEJS_18_X,
      handler: 'index.handler',
      code: lambda.Code.fromAsset('lambda'),
      environment: {
        BUCKET_NAME: bucket.bucketName
      }
    });

    // 3️⃣ 授权 Lambda 访问 S3
    bucket.grantReadWrite(fn);

    // 4️⃣ 创建 API Gateway
    new apigw.LambdaRestApi(this, 'MyApi', {
      handler: fn
    });
  }
}
```

# 🪜 Step 4：创建 Lambda 代码

新建文件夹：`lambda/`
文件：`lambda/index.js`

```js
const AWS = require('aws-sdk');
const s3 = new AWS.S3();

exports.handler = async (event) => {
    const bucket = process.env.BUCKET_NAME;

    const params = {
        Bucket: bucket,
        Key: `test-${Date.now()}.txt`,
        Body: "Hello from Lambda!"
    };

    await s3.putObject(params).promise();

    return {
        statusCode: 200,
        body: JSON.stringify({
            message: "File uploaded to S3!"
        })
    };
};
```

# 🧠 Step 2：Lambda 代码（核心升级）

文件：`lambda_function.py`

```python
import json
import boto3
import os

s3 = boto3.client('s3')

def lambda_handler(event, context):
    bucket = os.environ['BUCKET_NAME']
    key = 'balances.json'

    # 1️⃣ 获取 query 参数
    user_id = event.get('queryStringParameters', {}).get('userId')

    if not user_id:
        return {
            "statusCode": 400,
            "body": json.dumps({"error": "Missing userId"})
        }

    # 2️⃣ 从 S3 读取数据
    response = s3.get_object(Bucket=bucket, Key=key)
    data = json.loads(response['Body'].read())

    # 3️⃣ 查询余额
    balance = data.get(user_id)

    if balance is None:
        return {
            "statusCode": 404,
            "body": json.dumps({"error": "User not found"})
        }

    # 4️⃣ 返回结果
    return {
        "statusCode": 200,
        "body": json.dumps({
            "userId": user_id,
            "balance": balance
        })
    }
```

# 🟢 CDK 示例：VPC + EC2 Web Server

## 一、安装依赖（如果还没做）

```bash
npm install aws-cdk-lib constructs
```

---

## 二、完整代码（Stack）

```ts
import * as cdk from 'aws-cdk-lib';
import { Stack, StackProps } from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as ec2 from 'aws-cdk-lib/aws-ec2';

export class WebServerStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    // 1️⃣ 创建 VPC
    const vpc = new ec2.Vpc(this, 'MyVPC', {
      maxAzs: 2
    });

    // 2️⃣ 创建 Security Group（允许 HTTP）
    const sg = new ec2.SecurityGroup(this, 'WebSG', {
      vpc,
      allowAllOutbound: true,
    });

    sg.addIngressRule(
      ec2.Peer.anyIpv4(),
      ec2.Port.tcp(80),
      'Allow HTTP access'
    );

    // 3️⃣ 创建 EC2 实例
    const instance = new ec2.Instance(this, 'WebServerInstance', {
      vpc,
      instanceType: ec2.InstanceType.of(
        ec2.InstanceClass.T2,
        ec2.InstanceSize.MICRO
      ),
      machineImage: ec2.MachineImage.latestAmazonLinux(),
      securityGroup: sg,
      vpcSubnets: {
        subnetType: ec2.SubnetType.PUBLIC, // 放在公有子网
      },
    });

    // 4️⃣ 添加 User Data（自动安装 Web Server）
    instance.addUserData(
      `#!/bin/bash
      yum update -y
      yum install -y httpd
      systemctl start httpd
      systemctl enable httpd
      echo "<h1>Hello from CDK Web Server 🚀</h1>" > /var/www/html/index.html`
    );
  }
}
```

# 🧩 AWS CDK v2 – VPC 参数讲解

## 🔹 1. ipAddresses（IP 地址范围）

👉 用来定义 VPC 的 CIDR 网段

示例：

```
10.0.0.0/16
```

含义：

* `/16` 表示子网掩码
* 总 IP 数量 ≈ 65,536 个（约 6.5 万）

📌 计算方式：

```
2^(32 - 16) = 65536
```

👉 这个范围会被用来划分子网（public / private）

---

## 🔹 2. vpcName（VPC 名称）

👉 给你的 VPC 起名字（可选）

示例：

```
vpcName: 'my-vpc'
```

📌 作用：

* 方便在 AWS 控制台识别
* 不影响功能

---

## 🔹 3. natGateways（NAT 网关数量）⚠️重点

👉 控制是否创建 NAT Gateway

示例：

```
natGateways: 0
```

📌 为什么重要：

* NAT Gateway **是收费的**
* 每个 NAT Gateway 都会产生费用 💰

👉 设置为 0：

* ❌ 不创建 NAT Gateway
* 💰 避免额外费用（非常适合学习/测试）

---

# 🧠 总结（面试/考试重点）

| 参数          | 作用           | 推荐               |
| ----------- | ------------ | ---------------- |
| ipAddresses | 定义 VPC IP 范围 | 常用 `10.0.0.0/16` |
| vpcName     | VPC 名称       | 可选               |
| natGateways | NAT 数量       | 👉 学习环境设为 `0`    |

---

# 💻 CDK 示例代码（TypeScript）

```ts
import * as ec2 from 'aws-cdk-lib/aws-ec2';

const vpc = new ec2.Vpc(this, 'MyVPC', {
  ipAddresses: ec2.IpAddresses.cidr('10.0.0.0/16'),
  vpcName: 'my-vpc',
  natGateways: 0
});
```

* 否则：
  👉 `natGateways: 0` 是最省钱方案

# 🔐 AWS CDK – Security Group 讲解

## 🧩 什么是 Security Group？

👉 可以理解为 **EC2 的防火墙**

* 控制：

  * ✅ 哪些流量可以进入（Inbound）
  * ✅ 哪些流量可以出去（Outbound）

---

# 📌 Props 参数说明

## 🔹 1. vpc（必须）

👉 指定这个安全组属于哪个 VPC

```ts
vpc: vpc
```

---

## 🔹 2. allowAllOutbound（是否允许所有出站流量）

```ts
allowAllOutbound: true
```

📌 含义：

* `true`（默认）：允许访问互联网 🌐
* `false`：完全禁止出站（高级用法）

👉 一般保持 `true`

---

## 🔹 3. description（描述，可选）

```ts
description: 'Allow HTTP access'
```

👉 用来说明用途（建议写）

---

## 🔹 4. securityGroupName（名称，可选）

```ts
securityGroupName: 'web-sg'
```

👉 方便在控制台识别

---

## 🔹 5. addIngressRule（重点🔥）

👉 添加**入站规则（开放端口）**

```ts
addIngressRule(peer, connection, description?)
```

### 常见用法：

👉 开放 HTTP（80端口）

```ts
sg.addIngressRule(
  ec2.Peer.anyIpv4(),
  ec2.Port.tcp(80),
  'Allow HTTP'
);
```

👉 开放 SSH（22端口）

```ts
sg.addIngressRule(
  ec2.Peer.anyIpv4(),
  ec2.Port.tcp(22),
  'Allow SSH'
);
```

# 🧠 总结（必须记住）

| 参数                | 作用     | 是否重要      |
| ----------------- | ------ | --------- |
| vpc               | 绑定 VPC | ⭐⭐⭐       |
| allowAllOutbound  | 控制出站   | ⭐⭐        |
| description       | 描述     | ⭐         |
| securityGroupName | 名称     | ⭐         |
| addIngressRule    | 开放端口   | ⭐⭐⭐⭐（最重要） |

# 💻 完整 CDK 示例（Security Group）

```ts
import * as ec2 from 'aws-cdk-lib/aws-ec2';

const sg = new ec2.SecurityGroup(this, 'WebSG', {
  vpc: vpc,
  allowAllOutbound: true,
  description: 'Security group for web server',
  securityGroupName: 'web-sg'
});

// 允许 HTTP
sg.addIngressRule(
  ec2.Peer.anyIpv4(),
  ec2.Port.tcp(80),
  'Allow HTTP'
);

// 允许 SSH
sg.addIngressRule(
  ec2.Peer.anyIpv4(),
  ec2.Port.tcp(22),
  'Allow SSH'
);
```

# ⚠️ 常见坑（考试/面试会问）

❌ 没开 80端口 → 浏览器访问不了网站
❌ 没开 22端口 → SSH 连不上
❌ 过度开放（0.0.0.0/0） → 安全风险

👉 实际生产建议：

* SSH 只允许你的 IP
* HTTP/HTTPS 才开放公网

# 💻 完整组合代码（强烈建议记住结构）

```ts
import * as ec2 from 'aws-cdk-lib/aws-ec2';

const instance = new ec2.Instance(this, 'WebServer', {
  vpc: vpc,

  vpcSubnets: {
    subnetType: ec2.SubnetType.PUBLIC
  },

  securityGroup: sg,

  instanceType: ec2.InstanceType.of(
    ec2.InstanceClass.T2,
    ec2.InstanceSize.MICRO
  ),

  machineImage: ec2.MachineImage.latestAmazonLinux(),

  keyName: 'my-keypair'
});

instance.addUserData(
  '#!/bin/bash',
  'yum update -y',
  'yum install -y nginx',
  'systemctl start nginx',
  'echo "Hello World" > /usr/share/nginx/html/index.html'
);
```

# ⚠️ 常见错误（非常重要）

❌ 没设置 PUBLIC subnet → 无法访问
❌ 没开 80端口 → 页面打不开
❌ keyName 不存在 → 部署失败
❌ userData 写错 → 服务没启动
