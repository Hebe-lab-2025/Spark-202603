### 🧠 一句话核心（面试版）

👉 **CDK = 用代码（Java / Python / TS）定义云资源 → 自动生成 CloudFormation → 一键部署到 AWS**

---

## 📦 CDK 到底在干嘛？

传统方式：

* 你要建 AWS 资源（EC2、Lambda、S3）
* 写一堆 **YAML / JSON（CloudFormation）**
* 很难维护 😓

👉 CDK 的作用：

* 用你熟悉的编程语言写
* 自动帮你生成底层模板
* 帮你部署

---

## ⚙️ 工作流程（一定要会说）

👉 面试高频流程：

1. 写代码（CDK App）
2. 定义资源（Stack）
3. `cdk synth`
   → 生成 CloudFormation 模板
4. `cdk deploy`
   → 部署到 AWS

---

## 🧱 核心概念（非常重要）

### 1️⃣ App（应用）

👉 整个 CDK 项目入口

---

### 2️⃣ Stack（栈）

👉 一组资源（对应一个 CloudFormation Stack）

例子：

* 一个 Web App
* 一个 Backend 服务

---

### 3️⃣ Construct（构造）

👉 最小单位（资源组件）

分三层：

| 层级 | 含义             |
| -- | -------------- |
| L1 | 原生 AWS 资源（最底层） |
| L2 | 封装好的资源（最常用）    |
| L3 | 高级组合（架构级）      |

---

## 🧑‍💻 简单例子（Java风格理解）

```java
Bucket bucket = Bucket.Builder.create(this, "MyBucket")
    .versioned(true)
    .build();
```

👉 你写的是“业务代码”，不是配置文件

---

## 🔥 为什么大家用 CDK？

### ✅ 优点（面试要背）

* 用代码 → 可复用、可测试
* 支持 OOP（封装 / 继承）
* 自动生成 CloudFormation
* 易读性高
* 更适合复杂系统

---

## ⚠️ 和 CloudFormation 区别

| CDK | CloudFormation |
| --- | -------------- |
| 写代码 | 写 YAML         |
| 可编程 | 静态配置           |
| 易维护 | 难维护            |

👉 本质：
👉 **CDK = CloudFormation 的“高级封装”**

---

## 🚀 常见使用场景

* 部署 Lambda + API Gateway
* 构建微服务架构
* 自动化基础设施（IaC）
* DevOps / CI-CD pipeline

---

## 💡 面试加分总结（一句话）

**“CDK lets us define infrastructure using programming languages, which improves maintainability, 
**reusability, and scalability compared to raw CloudFormation.”**

## ✅ 必备环境（按顺序）

### 1️⃣ AWS账号

👉 用来创建云资源（必须）

---

### 2️⃣ IDE（推荐 VS Code）

👉 写 CDK 代码

---

### 3️⃣ AWS CLI（核心工具）

👉 本地和 AWS 通信

```bash
aws --version
```

---

### 4️⃣ 安装 CDK Toolkit

```bash
npm install -g aws-cdk
```

验证：

```bash
cdk --version
```

---

### 5️⃣ Node.js（必须）

👉 CDK 底层依赖

```bash
node --version
```

---

### 6️⃣ 配置 AWS 凭证（非常关键🔥）

```bash
aws configure
```

输入：

* Access Key
* Secret Key
* region（比如 us-west-2）

验证：

```bash
aws s3 ls
```

👉 能看到 bucket = 成功

## 📈 发展过程（一定要会说）

### 1️⃣ 手动（Console）

* 点网页创建资源
* ❌ 不可复用
* ❌ 容易出错

---

### 2️⃣ CLI / 脚本

* 用命令行创建
* ❌ 还是不够结构化

---

### 3️⃣ CloudFormation（声明式）

* 写 YAML / JSON
* ✅ 支持 rollback
* ✅ 支持 CI/CD
* ❌ 难维护

---

### 4️⃣ CDK（命令式 + 编程化🔥）

👉 用代码写 infra

---

# 🧠 三、为什么 CDK 更强（重点）

## ✅ 优势

* 支持 OOP（封装 / 继承）
* 代码复用
* 更少代码（比 YAML 少很多）
* 更容易维护
* 可以写逻辑（if / loop）

---

## ⚠️ 缺点

* 需要学习（Node / CDK）
* 还在 evolving

---

# 🧠 四、你要真正理解的本质（非常重要）

👉 CDK 并不是直接创建资源

👉 它做的是：

```text
CDK code
   ↓
CloudFormation template
   ↓
AWS resource
```
